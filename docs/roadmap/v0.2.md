# codemap v0.2 Roadmap

本文档记录 `codemap v0.2` 的目标、阶段边界和兼容性约束。v0.2 基于 v0.1.0 已完成的本地静态调用图闭环继续演进，不改变 v0.1 已有 CLI/API/Graph JSON Schema 的基础语义。

## v0.1.0 Baseline

v0.1.0 已完成：

- 扫描本地 Go module / Go workspace。
- 默认忽略 `_test.go`。
- 提取函数与方法 symbol，并生成稳定 symbol ID。
- 基于 AST 和类型信息提取函数调用、方法调用、interface 调用、external 调用和 unresolved 调用。
- 支持 `scan`、`symbols`、`calls`、`graph`、`serve` CLI。
- 构建从入口 symbol 出发的 depth-limited call graph。
- 默认隐藏标准库和第三方调用。
- 提供本地 HTTP API：`/api/health`、`/api/symbols`、`/api/graph`、`/api/source`、`/api/warnings`。
- 使用 Next.js、React、TypeScript、Tailwind CSS 和 React Flow 展示调用图。
- 点击图节点后通过 `/api/source` 懒加载源码片段。
- Go 二进制可以服务 API 和静态 Web UI。
- `make check` 覆盖 Go 测试、CLI smoke、golden 校验、Web lint、TypeScript check 和 Next build。

## v0.1.0 Limits

v0.1.0 保持以下限制：

- 只支持本地 Go 项目，不支持远程 GitHub URL 扫描。
- 不引入 LLM 解释代码。
- 不提供 VSCode、Zed 或 JetBrains 插件。
- 不依赖数据库。
- 不默认分析 `_test.go`。
- interface 调用只保守记录为 interface resolution，不保证展开到具体实现。
- 函数变量、复杂动态分派等调用可能记录为 unresolved。
- 调用图是静态近似结果，不承诺运行时精确调用链。
- 默认业务图隐藏标准库和第三方依赖调用。
- 源码全文不进入 graph JSON，源码片段通过 API 懒加载。

## v0.2 Goals

v0.2 聚焦提升代码理解能力和产品可用性：

1. 改进 interface call 的候选实现解析。
2. 增加 CLI/API/UI 层面的图过滤能力。
3. 优化入口 symbol 搜索和 package 导航。
4. 改进 React Flow 布局、选择态、交互反馈和可读性。
5. 支持 edge callsite 源码查看。
6. 增加性能指标、缓存刷新和项目元信息。
7. 完善 CI、release、demo 文档和 v0.2 质量门禁。

## Compatibility Contract

v0.2 必须保持 v0.1 的兼容性：

- 不删除 `nodes`、`edges`、`warnings` 等既有 graph 字段。
- 不改变既有字段含义。
- 不让 v0.1 CLI 命令失效。
- Graph JSON Schema 只允许向后兼容的 optional 字段扩展。
- 新增 API query 参数或 endpoint 必须保持默认行为兼容。
- 不让 `make check` 依赖不可复现的外部状态。
- golden outputs 用于捕获 v0.1 行为回归；行为变化必须在对应阶段明确说明并同步更新文档。

## Stages

v0.2 按以下顺序推进：

1. Stage 09：Baseline and Compatibility Contract。
2. Stage 10：Interface Implementation Resolution。
3. Stage 11：Graph Filters in CLI and API。
4. Stage 12：Web Filter Controls and Symbol Navigation。
5. Stage 13：Graph Layout and Interaction Polish。
6. Stage 14：Source Viewer and Callsite Inspection。
7. Stage 15：Performance, Cache, Rescan, and Metadata。
8. Stage 16：CI, Release, and v0.2 Demo Docs。
9. Stage 17：v0.2 Quality Gate。
