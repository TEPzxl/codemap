# Stage 09 — v0.2 Baseline and Compatibility Contract

## 目标

在开始 v0.2 功能开发前，固定 v0.1.0 基线，建立兼容性约束、golden outputs 和回归检查入口。

本阶段不新增功能。

## 允许修改

- `docs/roadmap/v0.2.md`
- `docs/examples/` 或 `docs/golden/`
- `scripts/`
- `Makefile`
- README 中的 v0.2 说明

## 禁止修改

- `internal/analyzer/` 核心逻辑
- `internal/graph/` 核心逻辑
- `internal/server/` handler 行为
- `web/` 业务 UI
- Graph JSON Schema

## 需要完成

1. 创建 v0.2 roadmap 文档。
2. 记录 v0.1 当前能力和限制。
3. 生成至少三个 fixture 的 golden graph 输出：
   - `examples/simple`
   - `examples/layered-service`
   - `examples/interface-call`
4. 增加一个脚本，例如：

```bash
scripts/generate_golden.sh
```

用于生成或刷新 golden outputs。

5. 增加一个脚本，例如：

```bash
scripts/verify_golden.sh
```

用于验证当前 CLI 输出与 golden 是否一致。

6. 如果 golden 输出中存在不稳定字段，应先稳定排序，而不是在测试里忽略大量字段。

## 推荐 golden 输出

建议保存：

```text
docs/golden/v0.1/simple.graph.json
docs/golden/v0.1/layered-service.graph.json
docs/golden/v0.1/interface-call.graph.json
docs/golden/v0.1/simple.symbols.json
docs/golden/v0.1/layered-service.calls.json
```

## 验收命令

```bash
git status --short
make check
./scripts/generate_golden.sh
./scripts/verify_golden.sh
```

## 成功标准

- `make check` 通过。
- golden 生成脚本可运行。
- golden 校验脚本可运行。
- v0.1 的 `scan` / `symbols` / `calls` / `graph` / `serve` 命令未破坏。
- 没有改动 Graph JSON Schema。

## 常见失败模式

- golden JSON 中字段顺序不稳定。
- 图中 nodes / edges 顺序不稳定，导致每次输出不同。
- 为了通过校验修改核心逻辑。
- 把本阶段变成 v0.2 功能开发。
