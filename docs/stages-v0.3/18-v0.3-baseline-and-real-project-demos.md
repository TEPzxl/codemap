# Stage 18 — v0.3 Baseline and Real Project Demos

## 目标

固定 `v0.2.0` 基线，并把真实项目分析结果纳入文档体系。该阶段不新增功能，只为 `v0.3` 建立清晰起点。

## 背景

`v0.2.0` 已经可以扫描真实项目，例如：

- `contentflow`：28 packages、345 symbols、1572 calls、0 warnings。
- `raft-kv-extended`：7 packages、161 symbols、753 calls、0 warnings。

这些结果说明 `codemap` 已经从 fixture demo 进入真实项目可用阶段。`v0.3` 应围绕“大图探索体验”继续推进。

## 允许修改范围

可以修改：

- `README.md`
- `docs/demo/*`
- `docs/releases/*`
- `docs/design/*`
- `docs/stages-v0.3/*`

不要修改：

- `internal/analyzer/*`
- `internal/graph/*`
- `internal/server/*`
- `web/*`
- Graph JSON Schema

## 需要完成

1. 确认 `v0.2.0` tag 存在，或记录当前 v0.2 基线 commit。
2. 新增 `docs/demo/real-projects.md`。
3. 在文档中记录真实项目扫描结果：
   - 项目名称。
   - packages 数量。
   - symbols 数量。
   - calls 数量。
   - warnings 数量。
   - 使用的 entry symbol。
   - 观察到的问题，例如大图密度、main 入口展开过大。
4. 如果已经有截图，保存到：
   - `docs/demo/contentflow-callgraph.png`
   - `docs/demo/raft-kv-extended-callgraph.png`
5. README 增加 `Real Project Demo` 小节，链接到上述文档。
6. 明确 `v0.3` 主题：Focused Graph Exploration。

## 验收命令

```bash
git status --short
make check
make web-build
make build
```

## 成功标准

- v0.2 基线清晰。
- README 能看到真实项目 demo 入口。
- `docs/demo/real-projects.md` 记录了至少两个真实项目分析结果。
- 没有修改代码逻辑。
- `make check` 通过。

## 常见失败模式

- 只放截图，不记录 packages/symbols/calls/warnings 数据。
- 文档夸大能力，例如把静态候选说成运行时真实调用链。
- 在本阶段误改 analyzer 或 graph 逻辑。
