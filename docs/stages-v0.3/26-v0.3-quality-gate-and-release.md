# Stage 26 — v0.3 Quality Gate and Release

## 目标

对 v0.3 全部新增能力进行质量门禁、回归测试、文档一致性检查和 release 准备。

## 不新增大功能

本阶段只允许：

- 修复 bug。
- 补测试。
- 补文档。
- 修 Makefile/scripts。
- 调整 README 和 release notes。
- 小范围 UI 文案修复。

禁止：

- 新增大型功能。
- 重构 analyzer。
- 改 Graph JSON Schema。
- 引入 LLM 或远程 GitHub 扫描。

## 需要检查的功能

- v0.2 基本功能仍然通过。
- graph summary 和 URL view state。
- focus mode：downstream/upstream/both。
- path search。
- package graph。
- entrypoint discovery。
- large graph UI search/highlight/sidebar。
- export JSON/Mermaid/DOT。
- source/callsite inspection。
- interface candidate expansion。
- make check/build/release/smoke。

## Makefile 要求

至少包含：

```text
make check
make smoke
make web-lint
make web-typecheck
make web-build
make build
make release
```

`make check` 至少覆盖：

- `go test ./...`
- smoke script
- web lint
- web typecheck
- web build

## 文档要求

新增或更新：

- `docs/releases/v0.3.0.md`
- README 的 v0.3 功能说明
- demo 截图或 real project demo 文档
- Known limitations
- Upgrade notes

必须明确：

- 仅支持本地 Go repo。
- 默认不分析 `_test.go`。
- interface expansion 是静态候选。
- path search 是静态调用图路径，不代表运行时必经路径。
- package graph 是 call graph 聚合，不是 import graph。

## 验收命令

```bash
make check
make web-build
make build
make release
./scripts/smoke.sh
```

启动 demo：

```bash
./bin/codemap serve ./examples/layered-service --port 8080
```

验证 API：

```bash
curl -s http://localhost:8080/api/health
curl -s http://localhost:8080/api/meta
curl -s "http://localhost:8080/api/graph?entry=main.main&depth=5"
curl -s "http://localhost:8080/api/graph?entry=main.main&depth=3&direction=upstream"
curl -s "http://localhost:8080/api/path?from=main.main&to=UserRepository.Save&max_depth=8"
curl -s "http://localhost:8080/api/package-graph?entry=main.main&depth=5"
curl -s "http://localhost:8080/api/entrypoints"
curl -s "http://localhost:8080/api/export?entry=main.main&depth=5&format=mermaid"
```

## 成功标准

- 所有质量门禁通过。
- v0.3 release note 完成。
- Web UI 可打开，新增功能可手动验证。
- 没有提交 `bin/codemap` 或 `node_modules`。
- 若提交 `internal/server/static/`，必须是最新 web-build 产物。
- 可以打 `v0.3.0` tag。

## 常见失败模式

- 新增功能通过，但 v0.2 行为回归。
- README 命令和实际命令不一致。
- web-build 后忘记提交 static 产物。
- export/path/focus API 返回裸文本错误。
- v0.3 文档宣称运行时准确调用链。
