# codemap v0.2.0 发布说明

## Release Summary

`v0.2.0` 将 `codemap` 从 v0.1 的端到端基础闭环升级为一个更可用的本地代码导航工具。本版本保持原有的保守静态分析契约，同时改进了 interface call 可见性、图过滤、Web 导航、源码查看、缓存项目元信息、rescan 行为、CI、release 构建和 demo 文档。


默认行为仍与 v0.1 兼容：默认隐藏标准库和第三方调用，默认隐藏 unresolved 调用，并且只有显式启用时才展示接口实现候选。

## Major Features

### 接口实现候选展开

- 新增静态 interface implementation candidate expansion。
- CLI flag：`--expand-interface`。
- API query：`expand_interface=true`。
- 默认行为保持保守，不展开候选实现。
- 接口实现判断基于 Go type information，而不是字符串猜测。

### 统一图过滤

- 统一 CLI、API 和 Web UI 的 graph filtering options。
- 支持的过滤项包括：
  - external calls
  - unresolved calls
  - interface calls
  - interface candidate expansion
  - package prefix
  - node limit
- 现有 v0.1 graph 默认行为保持不变。

### Web UI 导航与过滤

- 新增入口 symbol 搜索。
- 新增 package filtering。
- 新增 graph filter toggles。
- 新增带 debounce 的 depth 控制自动刷新。
- 保留 `Load graph`，用于手动重试和手动 entry 加载。
- 改进 selected node 和 selected edge 高亮。

### 图布局与交互

- 改进 React Flow 自动布局。
- 新增 fit-view、minimap、reset layout 和更清晰的节点/边样式。
- 视觉样式现在能区分 function/method/external/unresolved 节点，以及 resolved/interface/external/unresolved 边。

### 源码查看与调用点检查

- 保留 v0.1 通过 `/api/source` 查看节点源码的行为。
- 新增通过 `/api/callsite` 查看边对应调用点源码。
- Web UI 现在可以显示：
  - `Node source`
  - `Callsite`
- callsite snippet 包含上下文，并高亮调用发生行。
- 源码文件读取仍限制在被分析项目 root 内。

### 项目元信息与 Rescan

- 新增 `/api/meta`，用于返回缓存分析元信息：
  - root
  - module
  - package count
  - symbol count
  - call count
  - warning count
  - analyzed timestamp
  - analysis duration
  - version
- 新增 `POST /api/rescan`。
- rescan 会重建新的项目索引，并在成功后原子替换旧索引。
- rescan 失败时返回 JSON error，并保留之前可用的索引。

### CI、Release 与 Demo 文档

- 新增 GitHub Actions CI workflow。
- CI 运行：
  - `make check`
  - `make web-build`
  - `make build`
- 新增 `make release`。
- release 产物：
  - `dist/codemap-linux-amd64`
  - `dist/codemap-darwin-arm64`
  - `dist/codemap-darwin-amd64`
- 新增 `docs/demo/` 下的可复现 demo 文档。

## Validation Commands

v0.2 质量门禁使用以下命令：

```bash
git status --short
make check
make web-build
make build
make release
./scripts/smoke.sh
```

对 race 敏感的包使用以下命令验证：

```bash
go test -race -count=1 ./internal/server/... ./internal/analyzer/... ./internal/graph/...
```

Demo server 验证：

```bash
./bin/codemap serve ./examples/layered-service --port 8080
curl -s http://localhost:8080/api/health
curl -s http://localhost:8080/api/meta | python -m json.tool
curl -s "http://localhost:8080/api/graph?entry=main.main&depth=5" | python -m json.tool
curl -s "http://localhost:8080/api/graph?entry=main.main&depth=5&show_external=true" | python -m json.tool
curl -s "http://localhost:8080/api/graph?entry=main.main&depth=5&expand_interface=true" | python -m json.tool
```

callsite 验证需要从 graph response 中取真实 edge id：

```bash
curl -s "http://localhost:8080/api/callsite?edge_id=<real_edge_id>&entry=main.main&depth=5" | python -m json.tool
```

interface candidate 验证：

```bash
./bin/codemap graph ./examples/interface-call --entry main.main --depth 5 --expand-interface
```

预期的 demo 观察结果：

- `/api/health` 返回 `{"status":"ok"}`。
- `/api/meta` 返回被分析项目的元信息。
- 默认 layered-service graph 显示本地业务调用链。
- `show_external=true` 会显示 external edges。
- API 和 CLI 都接受 `expand_interface=true` / `--expand-interface`。
- interface-call fixture 在启用 expansion 时包含 candidate edges。
- `/api/callsite` 对真实 graph edge 返回源码片段。

## Known Limitations

- 只支持本地 Go 项目。
- 不支持远程 GitHub URL 扫描。
- 默认忽略 `_test.go` 文件。
- interface expansion 展示的是静态候选，不保证等同于运行时分派。
- 动态函数值和 reflection 会被保守处理。
- 不支持字段级读写引用分析。
- 不支持变量级引用分析。
- 调用图是静态近似结果，不是运行时精确调用链。
- 标准库和第三方调用默认隐藏，除非显式启用。
- 不包含数据库持久化。
- 不包含 IDE 插件。
- 不包含 LLM 解释层。

## Next Roadmap

`v0.2.0` 之后建议继续推进：

1. 在不改变保守默认契约的前提下，提高 unresolved calls 的精度和可解释性。
2. 改进 package load failure 和 partial graph 的诊断信息。
3. 增强 callsite 附近的源码导航，例如 caller/callee jump 和 source panel history。
4. 在保持 local-first 模型的前提下，扩展 graph export 和分享工作流。
5. 增加更大的真实项目 fixture 覆盖和性能 benchmark。
6. 改进 release packaging，增加 checksums 和可选 Windows artifacts。
7. 基于现有 manual rescan API，考虑增加本地文件变化的 watch mode。
8. 后续 schema 变更继续保持对现有 Graph JSON consumers 的向后兼容。
