# codemap v0.3.0 发布说明

## Release Summary

`v0.3.0` 的主题是 **Focused Graph Exploration**。本版本在 v0.2 的本地静态调用图基础上，补齐了大图探索所需的入口发现、节点聚焦、路径查询、package 级概览、当前图搜索和导出分享能力。

本版本保持保守静态分析契约：

- 不改变 Graph JSON Schema 的必填字段。
- 不改变 v0.2 `/api/graph` 的默认 downstream 行为。
- 不引入远程 GitHub 扫描、数据库或 LLM 解释层。
- 不承诺运行时精确调用链，只展示静态可解析或保守记录的调用关系。

## Major Features

### Real Project Baseline

- 新增真实项目 demo baseline 文档。
- `contentflow` baseline：28 packages、345 symbols、1572 calls、0 warnings。
- `raft-kv-extended` baseline：7 packages、161 symbols、753 calls、0 warnings。
- 明确 v0.3 聚焦真实项目中的大图探索体验。

### Graph Summary 与 URL View State

- Web UI 新增 `Current graph` summary，展示 nodes、edges、packages 和 resolution counts。
- `entry`、`depth`、`direction`、filters 和 package filter 会同步到 URL query。
- 刷新页面后会恢复 URL 状态并重新加载 graph。
- 新增 `Copy view URL`。

### Focus Mode

- 支持从任意完整 symbol/node id 聚焦调用图。
- CLI flag：`--direction downstream|upstream|both`。
- API query：`direction=downstream|upstream|both`。
- Web UI 支持 `Focus downstream`、`Focus upstream`、`Focus neighborhood` 和 `Reset to entry`。
- traversal 保持 depth limit 和 cycle protection。

### Symbol Path Search

- 新增 `codemap path` 命令。
- 新增 `/api/path`。
- 使用 BFS 查找 `from` 到 `to` 的静态调用图路径，支持 `max_depth`、`limit` 和 cycle protection。
- Web UI 新增 `Path search` 面板，查询结果可切换为 path graph。

### Package-Level Call Overview

- 新增 `codemap packages` 命令。
- 新增 `/api/package-graph`。
- Package graph 从函数/方法 call graph 聚合，不使用 import graph 伪造调用关系。
- package node 展示相对 package 路径、symbol count 和 call count。
- package edge 展示 package-to-package call count。
- self edge 默认隐藏。

### Entrypoint Discovery

- 新增启发式 entrypoint discovery 模块。
- 新增 `codemap entrypoints` 命令。
- 新增 `/api/entrypoints`。
- entrypoint 包含 `id`、`label`、`package`、`file`、`kind` 和 `reasons`。
- Web UI 新增 `Entrypoints` 列表，点击候选后自动加载 graph。
- `main` functions 优先显示。

### Large Graph UI Polish

- Web UI 新增当前图搜索，覆盖 label、id、package 和 file。
- 点击搜索结果后会选中节点并移动 viewport 到该节点。
- 点击节点后高亮直接上游/下游节点和相关边。
- 新增 sidebar collapse，便于查看大图。
- 保持 UI 简洁，避免高饱和或过度配色。

### Export 与 Share

- 新增 `codemap export` 命令。
- 新增 `/api/export`，支持 `format=json|mermaid|dot`。
- 导出使用当前 `entry`、`depth`、`direction` 和 filter 参数。
- Mermaid 和 DOT 导出会转义特殊字符。
- Web UI 新增 `Export` 面板，支持 copy/download JSON、Mermaid 和 DOT。

## CLI 与 API Additions

新增 CLI：

```bash
codemap entrypoints <path>
codemap graph <path> --entry <symbol> --depth 5 --direction upstream
codemap path <path> --from <symbol> --to <symbol> --max-depth 8 --limit 5
codemap packages <path> --entry <symbol> --depth 5
codemap export <path> --entry <symbol> --depth 5 --format json|mermaid|dot
```

新增 HTTP API：

```text
GET /api/entrypoints
GET /api/graph?entry=<symbol>&depth=5&direction=downstream|upstream|both
GET /api/path?from=<symbol>&to=<symbol>&max_depth=8&limit=5
GET /api/package-graph?entry=<symbol>&depth=5
GET /api/export?entry=<symbol>&depth=5&format=json|mermaid|dot
```

## Validation Commands

v0.3 质量门禁使用以下命令：

```bash
git status --short
make check
make web-build
make build
make release
./scripts/smoke.sh
```

Demo server 验证：

```bash
./bin/codemap serve ./examples/layered-service --port 8080
curl -s http://localhost:8080/api/health
curl -s http://localhost:8080/api/meta
curl -s "http://localhost:8080/api/graph?entry=main.main&depth=5"
curl -s "http://localhost:8080/api/graph?entry=main.main&depth=3&direction=upstream"
curl -s "http://localhost:8080/api/path?from=main.main&to=UserRepository.Save&max_depth=8"
curl -s "http://localhost:8080/api/package-graph?entry=main.main&depth=5"
curl -s "http://localhost:8080/api/entrypoints"
curl -s "http://localhost:8080/api/export?entry=main.main&depth=5&format=mermaid"
```

预期的 demo 观察结果：

- `/api/health` 返回 `{"status":"ok"}`。
- `/api/graph` 默认返回 downstream function graph。
- `/api/graph?...&direction=upstream` 返回上游 focus graph，参数错误时返回 JSON error。
- `/api/path` 返回静态调用路径和对应 path graph。
- `/api/package-graph` 返回从函数/方法调用关系聚合的 package graph。
- `/api/entrypoints` 返回启发式入口候选和说明 note。
- `/api/export` 按 format 返回 JSON、Mermaid 或 DOT。

## Known Limitations

- 只支持本地 Go 项目。
- 不支持远程 GitHub URL 扫描。
- 默认忽略 `_test.go` 文件。
- interface expansion 展示的是静态候选，不保证等同于运行时分派。
- 动态函数值和 reflection 会被保守处理。
- 调用图是静态近似结果，不是运行时精确调用链。
- Path search 查询的是静态调用图路径，不代表运行时必经路径。
- Package graph 是 call graph 聚合结果，不是 import graph。
- Entrypoint discovery 是启发式推荐，不保证完整或绝对准确。
- 标准库和第三方调用默认隐藏，除非显式启用。
- 不包含数据库持久化。
- 不包含 IDE 插件。
- 不包含 LLM 解释层。

## Upgrade Notes

- v0.2 的 `/api/graph` 默认行为保持兼容，新增的 `direction` query 默认归一化为 `downstream`。
- v0.3 API 是 additive extension；Graph JSON Schema 必填字段没有变化。
- 若发布 Go binary，请先运行 `make web-build` 更新 `internal/server/static/`。
- `bin/`、`dist/` 和 `web/node_modules/` 是本地构建产物，不应进入 git。
